# 07_report_delete_flow.md

## 목적

추천 리포트 화면과 개인정보 삭제 플로우를 구현하기 위한 Codex 프롬프트입니다.

## 프롬프트

상담 리포트와 저장/삭제 플로우를 구현해줘.

프로젝트:

헤어샵 태블릿용 AI 상담 리포트 앱

목표:

얼굴 분석 결과와 추천 메뉴를 바탕으로 상담 리포트를 표시하고, 고객이 저장 또는 삭제를 선택할 수 있게 한다.

기술:

- React
- TypeScript
- Tailwind CSS
- 로컬 DB 레이어 사용

구현할 화면:

- ConsultationReportScreen
- FinalTreatmentConfirmScreen
- SaveOrDeleteScreen
- DeleteConfirmScreen
- DeleteCompleteScreen

리포트에 표시할 내용:

- 고객 선택 요약
- 얼굴 분석 요약
- 추천 스타일 방향
- 추천 시술 메뉴 2~3개
- 추천 이유
- 연결 추천 메뉴
- 예상 가격
- 예상 소요시간
- 디자이너 상담 문구
- AI 결과는 상담 참고용이라는 고지
- 실제 결과는 모발 상태와 디자이너 상담에 따라 달라질 수 있다는 고지

저장/삭제 정책:

- 저장 동의가 없으면 원본 사진, 분석 결과, 추천 결과, 리포트를 삭제한다.
- 삭제 후 삭제 로그를 남긴다.
- 삭제 로그에는 삭제 시각, 세션 ID, 삭제 항목, 삭제 사유만 남긴다.
- 민감한 얼굴 분석값은 삭제 로그에 남기지 않는다.
- 리포트 저장은 saveReportConsent가 true일 때만 가능하다.

주의:

- “시술 후 결과”라는 표현을 쓰지 않는다.
- “AI 스타일 참고 이미지” 또는 “상담 참고 자료”로 표현한다.
- 최종 시술 방향은 디자이너 상담 후 결정된다는 문구를 항상 표시한다.
- 삭제 후 뒤로가기로 리포트가 다시 보이면 안 된다.
- 삭제 완료 화면에서 삭제 항목을 명확히 보여준다.

완료 기준:

- 추천 리포트 화면 표시
- 최종 시술안 확인 화면 표시
- 저장/삭제 선택 가능
- 삭제 시 세션 데이터 삭제 함수 호출
- 삭제 완료 화면 표시
- 삭제 로그 생성
- TypeScript 에러 없음
