# 05_security_privacy_review.md

## 목적

얼굴 사진과 분석 데이터를 다루는 앱의 개인정보·보안 구조를 검토하기 위한 프롬프트입니다.

## 프롬프트

이 프로젝트는 헤어샵 태블릿에서 고객 얼굴 사진을 촬영하고 얼굴 분석 결과를 생성하는 앱이다.

개인정보 원칙:

- 고객 이름과 전화번호는 기본 수집하지 않는다.
- 원본 얼굴 사진은 저장 동의가 없으면 삭제한다.
- 분석 결과와 리포트도 저장 동의가 없으면 삭제한다.
- 리포트 저장, 발송, 재방문 보관은 별도 선택 동의로 분리한다.
- AI 결과는 실제 시술 결과를 보장하지 않는다.
- 실제 고객 사진을 외부 AI 도구에 업로드하지 않는다.
- 삭제 로그에는 민감한 얼굴 분석값을 저장하지 않는다.

현재 코드와 데이터 흐름을 검토하고 개인정보 리스크를 찾아줘.

검토 항목:

1. 원본 사진이 불필요하게 저장되는 지점
2. 분석 결과가 삭제되지 않을 가능성
3. 로컬 DB에 남는 민감 데이터
4. 브라우저 캐시나 임시 파일 위험
5. 삭제 로그에 과도한 정보가 남는지
6. 외부 API 전송 가능성
7. 동의 없이 저장되는 데이터
8. 클레임 방어 문구 누락 위치
9. 리포트 저장/발송 동의 분리 여부
10. 세션 종료 시 자동 삭제 가능성

## 출력 형식

1. 위험도 높은 문제
2. 중간 위험 문제
3. 낮은 위험 문제
4. 즉시 수정해야 할 항목
5. 코드 레벨 수정 제안
6. 화면 UX 수정 제안
7. 테스트 체크리스트
8. 개인정보 처리방침에 반영해야 할 항목
