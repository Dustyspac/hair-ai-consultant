# 04_refactoring.md

## 목적

코드가 커졌을 때 기능별 구조를 정리하고 유지보수성을 높이기 위한 프롬프트입니다.

## 프롬프트

현재 헤어샵 AI 상담 리포트 앱의 코드가 복잡해지고 있다.

리팩토링 목표:

- 고객 플로우, 관리자 플로우, 추천 엔진, 개인정보 동의, 리포트, 삭제 기능을 명확히 분리한다.
- TypeScript 타입을 일관성 있게 정리한다.
- 로컬 DB 접근 로직을 분리한다.
- UI 컴포넌트와 비즈니스 로직을 분리한다.
- 추천 로직을 테스트 가능하게 만든다.
- 개인정보 삭제 흐름이 누락되지 않게 한다.
- 태블릿 화면 기준 UI 구조를 유지한다.
- 나중에 PWA 또는 Capacitor로 전환하기 쉽게 만든다.

현재 코드베이스를 검토하고 리팩토링 계획을 제시해줘.

## 출력 형식

1. 현재 코드의 문제점
2. 리팩토링 우선순위
3. 추천 폴더 구조
4. 이동해야 할 파일
5. 분리해야 할 로직
6. 공통 타입 정리안
7. 테스트 추가가 필요한 영역
8. 리팩토링 순서
9. 리팩토링 후 검증 체크리스트
10. 리팩토링 중 건드리면 위험한 영역
