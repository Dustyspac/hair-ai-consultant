# Prompts Guide

이 폴더는 헤어샵 AI 상담 리포트 MVP 개발에 사용할 AI 프롬프트를 정리하는 공간입니다.

## 1. 사용 목적

이 프로젝트에서는 AI 도구를 역할별로 나누어 사용합니다.

### ChatGPT

기획, 문서화, 추천 규칙, 개인정보 문구, 영업자료 작성에 사용합니다.

주요 용도:

- PRD 정리
- MVP 범위 확정
- 사용자 플로우 설계
- 화면 요구사항 정리
- 개인정보/클레임 리스크 문구 작성
- 추천 규칙 작성
- 파일럿 테스트 계획 작성
- 헤어샵 영업 스크립트 작성

### Claude Code

코드베이스 검토, 아키텍처 리뷰, 리팩토링, 버그 분석, 개인정보 흐름 검토에 사용합니다.

주요 용도:

- 전체 코드 구조 검토
- 아키텍처 리뷰
- 버그 원인 분석
- 리팩토링 계획 수립
- 개인정보/삭제 흐름 검토
- 보안 리스크 점검
- 테스트 계획 검토

### Codex

실제 코드 생성, 컴포넌트 구현, CRUD, 로컬 DB, 추천 엔진 개발에 사용합니다.

주요 용도:

- React 프로젝트 세팅
- UI 컴포넌트 구현
- 관리자 CRUD 구현
- 로컬 DB 레이어 구현
- 추천 엔진 구현
- 촬영 화면 구현
- 상담 리포트 구현
- 삭제 플로우 구현
- 테스트 코드 작성

## 2. 기본 프로젝트 설명

아래 설명은 모든 AI 프롬프트의 시작 부분에 공통 맥락으로 사용합니다.

프로젝트명:

- 헤어샵 AI 상담 리포트 태블릿 앱

프로젝트 목표:

- 고객이 헤어샵에서 태블릿으로 시술 메뉴와 고민을 선택한다.
- 디자이너가 고객 동의를 확인한 뒤 촬영 가이드에 맞춰 얼굴 사진을 촬영한다.
- 얼굴형, 얼굴 비율, 중안부, 턱선, 광대 등 얼굴 구조를 분석한다.
- 분석 결과와 고객 니즈를 바탕으로 해당 헤어샵에 실제 등록된 시술 메뉴 중 적합한 조합을 추천한다.
- 추천 결과는 디자이너가 상담에 활용할 수 있는 리포트로 제공한다.
- 고객은 현장에서 리포트 저장 또는 개인정보 삭제를 선택할 수 있다.

핵심 원칙:

- 1차 MVP는 태블릿 로컬 우선 구조로 개발한다.
- 고객 이름과 전화번호는 기본 수집하지 않는다.
- 저장 동의가 없으면 원본 사진, 분석 결과, 리포트를 삭제한다.
- AI 결과는 실제 시술 결과를 보장하지 않는다.
- AI 결과는 디자이너 상담을 돕기 위한 참고 자료로만 사용한다.
- 추천은 반드시 매장에 등록된 실제 시술 메뉴와 연결되어야 한다.
- 1차 MVP에서는 정교한 AI 헤어 합성 기능은 제외한다.
- 고객의 실제 얼굴 사진을 외부 AI 도구에 업로드하지 않는다.

기술 스택:

- React
- TypeScript
- Vite
- Tailwind CSS
- 로컬 DB
- MediaPipe Face Landmarker 예정
- 추후 PWA 또는 Capacitor 검토

## 3. AI 사용 규칙

1. AI에게 한 번에 전체 앱을 만들라고 요청하지 않는다.
2. 기능 단위로 작게 나누어 요청한다.
3. 개인정보, 삭제, 클레임 문구는 사람이 반드시 검수한다.
4. 실제 고객 얼굴 사진을 외부 AI에 업로드하지 않는다.
5. Codex나 Claude Code가 수정한 코드는 직접 실행 테스트한다.
6. 코드 생성 전에 관련 문서를 먼저 읽게 한다.
7. 기능 구현 후 완료 기준을 기준으로 검수한다.
8. 임시 구현과 실제 운영 구현을 구분해서 기록한다.

## 4. 프롬프트 작성 규칙

좋은 프롬프트에는 아래 항목이 포함되어야 한다.

- 프로젝트 목적
- 현재 작업 목표
- 기술 스택
- 입력 데이터
- 출력 데이터
- 구현할 파일 또는 폴더
- 구현하지 말아야 할 기능
- 완료 기준
- 주의사항

나쁜 요청 예시:

- “앱 전체를 만들어줘.”
- “추천 기능 만들어줘.”
- “대충 화면 만들어줘.”

좋은 요청 예시:

- “src/features/admin 폴더에 TreatmentMenu CRUD 기능을 구현해줘. TreatmentMenu 타입은 src/types/index.ts를 사용하고, 메뉴명/카테고리/가격/소요시간/추천조건/비추천조건을 입력할 수 있어야 해. 실제 서버 연동은 하지 말고 로컬 상태 기반으로 구현해줘.”

## 5. 우선 사용 순서

1. ChatGPT로 기획 문서 정리
2. Codex로 프로젝트 초기 세팅
3. Codex로 화면 단위 구현
4. Claude Code로 구조 검토
5. Codex로 기능 구현
6. Claude Code로 버그 분석 및 리팩토링
7. 사람이 직접 태블릿에서 테스트
