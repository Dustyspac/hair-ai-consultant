# 07_BACKLOG.md

## 1. 문서 목적

이 문서는 **헤어샵 태블릿용 AI 상담 리포트 MVP**를 3개월 안에 개발하기 위한 개발 백로그를 정리한 문서입니다.

개발 기준은 다음과 같습니다.

- 1인 개발 기준
- React + TypeScript + Vite + Tailwind CSS 기반
- 로컬 DB 우선
- MediaPipe Face Landmarker 연동 예정
- 추후 PWA 또는 Capacitor 확장 가능
- 1차 목표는 실제 헤어샵 파일럿 테스트 가능 수준의 MVP 완성

---

## 2. MVP 범위

### 포함 기능

- 고객 시술 메뉴 선택
- 고객 고민 선택
- 원하는 이미지 선택
- 개인정보 동의
- 얼굴 촬영
- 얼굴 분석
- 폴리곤 스캔 UI
- 매장 실제 메뉴 기반 추천
- 상담 리포트
- 저장/삭제 선택
- 삭제 로그
- 관리자 메뉴 등록

### 제외 또는 후순위 기능

- 고객 회원가입
- 고객 이름/전화번호 기본 수집
- 서버 연동
- 클라우드 저장
- 결제 기능
- 예약 기능
- AI 이미지 고도화
- PDF 출력
- 문자/카카오톡 발송
- 다매장 관리
- 정식 PWA/Capacitor 패키징

## 7. 개발 순서

### 1단계: 프로젝트 기반 구축

우선 앱이 정상적으로 실행되고, 고객/관리자 화면이 분리되어야 합니다.

대상 작업:

- T-001 React 프로젝트 생성
- T-002 Tailwind CSS 설정
- T-003 라우팅 구조 생성
- T-004 기본 폴더 구조 생성
- T-005 공통 UI 컴포넌트 생성

완료 기준:

- 로컬에서 앱 실행 가능
- 고객 화면과 관리자 화면 이동 가능
- 공통 버튼, 카드, 모달 사용 가능

---

### 2단계: 관리자 메뉴 설정

추천 엔진은 매장에 등록된 실제 메뉴를 기반으로 해야 하므로, 관리자 메뉴 설정을 먼저 구현합니다.

대상 작업:

- T-015 관리자 홈 화면
- T-016 관리자 메뉴 CRUD
- T-017 가격/소요시간 입력
- T-018 메뉴 활성/비활성 처리
- T-019 추천 조건 등록
- T-020 샘플 메뉴 데이터 작성

완료 기준:

- 매장 메뉴 등록 가능
- 가격과 소요시간 입력 가능
- 추천 조건 태그 등록 가능
- 비활성 메뉴가 고객 화면과 추천 결과에서 제외됨

---

### 3단계: 고객 입력 플로우

고객이 상담에 필요한 기본 정보를 입력하는 단계입니다.

대상 작업:

- T-006 시작 화면 구현
- T-007 시술 메뉴 선택 화면
- T-008 고객 고민 선택 화면
- T-009 원하는 이미지 선택 화면
- T-010 상담 세션 상태 관리

완료 기준:

- 고객이 상담을 시작할 수 있음
- 시술 메뉴를 다중 선택할 수 있음
- 고민을 선택할 수 있음
- 선택값이 다음 화면까지 유지됨

---

### 4단계: 개인정보 동의

촬영과 분석 전에 반드시 동의 구조를 구현합니다.

대상 작업:

- T-011 개인정보 동의 화면
- T-012 촬영/분석 동의 분리
- T-013 저장 동의 분리
- T-014 AI 결과 고지 문구 표시

완료 기준:

- 필수 동의 없이는 촬영 화면으로 이동 불가
- 촬영/분석 동의와 저장 동의가 분리됨
- AI 결과가 실제 결과를 보장하지 않는다는 문구가 표시됨

---

### 5단계: 얼굴 촬영/분석

태블릿 카메라를 활용해 얼굴을 촬영하고 MediaPipe Face Landmarker를 통해 분석합니다.

대상 작업:

- T-021 촬영 화면 구현

## 13. MVP 완료 기준

MVP 완료 기준은 다음과 같습니다.

- 고객이 태블릿에서 상담을 시작할 수 있다.
- 고객이 시술 메뉴와 고민을 선택할 수 있다.
- 촬영/분석 동의를 하지 않으면 촬영 단계로 넘어갈 수 없다.
- 태블릿 카메라로 얼굴 사진을 촬영할 수 있다.
- 얼굴 랜드마크 분석이 가능하다.
- 분석 중 폴리곤 스캔 UI가 표시된다.
- 매장에 등록된 실제 메뉴만 추천된다.
- 추천 메뉴의 가격, 소요시간, 추천 이유가 표시된다.
- AI 분석 결과가 실제 결과를 보장하지 않는다는 문구가 표시된다.
- 저장 동의가 없으면 원본 사진, 분석 결과, 리포트가 삭제된다.
- 삭제 시 삭제 로그가 남는다.
- 관리자가 매장 메뉴를 등록, 수정, 비활성화할 수 있다.
- 실제 태블릿에서 상담 1건을 처음부터 끝까지 완료할 수 있다.

## 14. MVP 개발 시 주의사항

### 14.1 개인정보 수집 최소화

1차 MVP에서는 고객 이름과 전화번호를 기본 수집하지 않습니다.

수집하지 않는 데이터:

- 고객 이름
- 고객 전화번호
- 고객 생년월일
- 고객 주소
- 고객 SNS 계정

기본 상담 세션은 익명 ID 기준으로 관리합니다.

예시:

```ts
sessionId: "local_session_001";
```

---

### 14.2 저장 동의 전 데이터는 임시 데이터로 처리

저장 동의 전에는 다음 데이터를 영구 저장하지 않습니다.

- 원본 얼굴 사진
- 얼굴 랜드마크 원본 데이터
- 얼굴 분석 결과
- 추천 결과
- 상담 리포트

상담 종료 시 고객이 저장을 선택하지 않으면 관련 데이터를 삭제합니다.

---

### 14.3 AI 결과는 상담 참고용으로만 표현

사용 가능한 표현:

```txt
AI 분석 결과는 상담을 돕기 위한 참고 자료입니다.
실제 시술 결과는 고객의 모발 상태, 시술 이력, 디자이너 상담에 따라 달라질 수 있습니다.
```

피해야 할 표현:

```txt
AI가 정확히 진단했습니다.
이 스타일이 가장 완벽합니다.
실제 결과와 동일합니다.
이 시술을 하면 얼굴형이 보정됩니다.
```

---

### 14.4 추천은 반드시 실제 메뉴와 연결

추천 결과는 텍스트로만 생성하지 않고, 관리자 메뉴에 등록된 실제 메뉴 ID와 연결합니다.

좋은 예시:

```ts
recommendedMenuId: "menu_perm_c_curl_001";
```

나쁜 예시:

```ts
recommendedStyle: "레이어드 C컬펌";
```

---

### 14.5 삭제 기능은 반드시 직접 테스트

삭제 기능은 구현만으로 완료 처리하지 않습니다.

반드시 확인해야 할 항목:

- 원본 사진 삭제 여부
- 분석 결과 삭제 여부
- 추천 결과 삭제 여부
- 리포트 삭제 여부
- 삭제 로그 기록 여부
- 저장 미동의 상태에서 데이터가 남지 않는지 여부

4. 개인정보 동의
5. 얼굴 촬영

7. 추천 엔진
8. 상담 리포트

10. 삭제 로그
11. 태블릿 실기기 테스트

1차 MVP의 핵심은 기능을 많이 넣는 것이 아니라, 실제 헤어샵에서 고객 상담 1건을 안전하게 끝낼 수 있는 구조를 만드는 것입니다.
