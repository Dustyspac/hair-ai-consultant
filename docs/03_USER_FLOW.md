# 헤어샵 AI 상담 리포트 프로그램 사용자 플로우

## 1. 문서 목적

이 문서는 태블릿 기반 매장용 **헤어샵 AI 상담 리포트 프로그램**의 사용자 플로우를 정의한다.

본 서비스는 고객이 태블릿에서 시술 메뉴와 고민을 선택하고, 디자이너가 동의 확인 후 얼굴 사진을 촬영한 뒤, 얼굴 분석 결과와 매장 실제 시술 메뉴를 연결해 상담 리포트를 제공하는 프로그램이다.

AI 이미지와 분석 결과는 실제 시술 결과를 보장하지 않으며, 디자이너 상담을 보조하기 위한 참고 자료로만 사용한다.

---

## 2. 사용자 유형

| 사용자      | 역할                                                                         |
| ----------- | ---------------------------------------------------------------------------- |
| 고객        | 태블릿에서 관심 시술, 고민, 원하는 이미지를 선택하고 상담 리포트를 확인한다. |
| 디자이너    | 고객 동의 여부를 확인하고 사진 촬영, 분석 결과 확인, 최종 상담을 진행한다.   |
| 매장 관리자 | 매장 메뉴, 가격, 소요시간, 추천 조건, 상담 문구를 등록하고 관리한다.         |

---

## 3. 서비스 핵심 조건

- 고객은 태블릿에서 시술 메뉴와 고민을 선택한다.
- 디자이너는 개인정보 및 촬영 동의 확인 후 사진을 촬영한다.
- 얼굴 분석 후 매장 실제 메뉴 기반 추천 리포트가 생성된다.
- AI 이미지와 리포트는 실제 시술 결과 보장이 아니라 상담 참고용이다.
- 저장 동의가 없으면 상담 종료 후 사진, 분석 결과, 리포트는 삭제된다.
- 관리자는 매장 메뉴, 가격, 소요시간, 추천 조건을 등록한다.
- 고객 이름과 전화번호는 MVP 단계에서 기본적으로 수집하지 않는다.

# 4. 고객 플로우

## 4.1 고객 기본 흐름

```text
매장 태블릿 시작
→ 서비스 목적 안내
→ 개인정보 수집·이용 동의
→ AI 리포트 한계 고지 확인
→ 관심 시술 메뉴 선택
→ 현재 고민 선택
→ 원하는 이미지 선택
→ 디자이너에게 태블릿 전달
→ 사진 촬영 동의 확인
→ 얼굴 촬영
→ 분석 대기
→ 상담 리포트 확인
→ 디자이너 상담 진행
→ 저장 여부 선택
→ 저장 동의 시 리포트 저장 또는 전송
→ 저장 미동의 시 상담 종료 후 삭제
→ 종료 화면
```

## 4.3 고객 입력 항목

| 구분          | 입력값 예시                                                                                              |
| ------------- | -------------------------------------------------------------------------------------------------------- |
| 관심 시술     | 커트, 펌, 염색, 클리닉, 앞머리, 볼륨매직, 스타일링                                                       |
| 얼굴형 고민   | 얼굴이 길어 보여요, 얼굴이 둥글어 보여요, 광대가 도드라져 보여요, 턱선이 강해 보여요, 이마가 넓어 보여요 |
| 모발 고민     | 볼륨이 없어요, 머리가 잘 가라앉아요, 손상이 심해요, 관리가 어려워요, 스타일 유지가 안 돼요               |
| 원하는 이미지 | 부드러운, 세련된, 어려 보이는, 차분한, 고급스러운, 자연스러운                                            |
| 우선순위      | 디자인 변화, 관리 편의성, 손상 최소화, 가격, 시술 시간                                                   |
| 저장 선택     | 저장 안 함, 리포트 저장, 고객 전송                                                                       |

## 4.4 고객 화면 UX 원칙

고객에게는 AI가 얼굴을 평가한다는 느낌보다, 상담을 쉽게 하기 위한 사전 체크라는 인상을 줘야 한다.

### 권장 표현

```text
고객님의 고민과 원하는 이미지를 바탕으로 디자이너 상담에 참고할 리포트를 생성합니다.
```

### 피해야 할 표현

```text
AI가 얼굴 단점을 분석합니다.
가장 잘 어울리는 스타일을 정확히 찾아드립니다.
시술 결과를 미리 확인할 수 있습니다.
```

# 5. 디자이너 플로우

## 5.1 디자이너 기본 흐름

```text
디자이너 로그인 또는 PIN 입력
→ 고객 세션 선택
→ 고객 입력 내용 확인
→ 개인정보·촬영 동의 여부 확인
→ 촬영 가이드 안내
→ 고객 얼굴 촬영
→ 사진 품질 확인
→ 재촬영 또는 분석 진행
→ 얼굴 분석 결과 확인
→ 매장 메뉴 기반 추천 확인
→ 추천 사유 확인
→ 고객과 상담
→ 최종 제안 메뉴 선택
→ 예상 가격·소요시간 안내
→ 저장 또는 삭제 처리
→ 상담 종료
```

## 10.2 동의 항목 분리 방식


## 10.3 MVP 권장 동의 방식

MVP에서는 동의를 한 화면에 모두 몰아넣기보다 단계별로 분리한다.

```text
1차: 서비스 목적 안내
2차: 사진 촬영·분석 동의
3차: 저장 여부 선택
4차: 마케팅 활용 동의는 MVP에서 제외
```

마케팅 활용 동의는 초기 MVP에서 제외하는 것이 좋다.
초기 서비스의 핵심은 상담 전환과 리포트 검증이지 후기 광고 수집이 아니다.

# 11. 클레임 방어 고지가 필요한 지점

| 지점               | 고지 내용                                    |

```text
이 리포트는 고객 입력값과 촬영 이미지를 기반으로 생성된 상담 참고 자료입니다.
실제 시술 결과를 보장하지 않습니다.
```

### AI 이미지 하단

```text
AI 이미지는 스타일 방향을 이해하기 위한 예시입니다.
실제 색상, 컬 형태, 길이감, 질감, 손상도, 유지 기간과 다를 수 있습니다.
```

### 추천 메뉴 영역

```text
추천 메뉴는 매장 등록 메뉴와 고객 입력값을 기반으로 제안됩니다.
최종 시술 가능 여부는 디자이너의 모발 진단 후 결정됩니다.
```

### 가격·소요시간 영역

```text
표시된 가격과 소요시간은 기준 정보입니다.
모발 길이, 손상도, 추가 시술 여부에 따라 달라질 수 있습니다.
```

### 저장·전송 전

```text
저장 또는 전송되는 리포트는 상담 당시의 참고 자료입니다.
이후 모발 상태나 시술 조건에 따라 실제 상담 내용은 달라질 수 있습니다.
```

# 15. MVP 우선순위

이 서비스의 MVP는 단순한 AI 헤어스타일 앱이 아니라, **헤어샵 상담 객단가를 높이는 태블릿 상담 도구**로 설계해야 한다.

따라서 MVP 우선순위는 다음과 같다.

```text
1순위: 매장 실제 메뉴 기반 추천
2순위: 고객 고민 입력
3순위: 디자이너 상담 보조
4순위: 개인정보 삭제/저장 선택
5순위: AI 이미지 참고 자료
```

AI 이미지는 가장 눈에 띄는 기능이지만, 제품의 중심이 되면 클레임 위험이 커진다.
리포트의 중심은 **“왜 이 고객에게 이 매장 메뉴를 제안하는가”**가 되어야 한다.

# 16. 구현 시 주의사항

## 16.1 개인정보 관련 주의사항

- 저장 동의가 없으면 상담 종료 후 사진과 분석 결과를 삭제한다.
- 삭제 완료 여부를 로그로 남긴다.
- 원본 사진은 기본적으로 장기 저장하지 않는다.
- 고객 이름과 전화번호는 MVP에서 기본 수집하지 않는다.
- 마케팅 활용 동의는 상담 동의와 분리한다.
- 고객이 동의하지 않아도 기본 상담은 가능해야 한다.

---

## 16.2 클레임 방어 관련 주의사항

- AI 결과를 확정적 표현으로 보여주지 않는다.
- “가장 잘 어울림”, “정확한 예측”, “시술 후 모습” 같은 표현은 피한다.
- “상담 참고용”, “실제 결과와 다를 수 있음”, “최종 판단은 디자이너 상담 기준” 문구를 반복 노출한다.
- 가격과 소요시간은 확정값이 아니라 기준값으로 표현한다.
- AI 이미지는 실제 결과물이 아니라 스타일 방향 예시로 표시한다.

---

## 16.3 추천 로직 관련 주의사항

- 추천은 반드시 매장에 등록된 실제 메뉴와 연결되어야 한다.
- 존재하지 않는 메뉴를 AI가 임의로 추천하면 안 된다.
- 추천 결과에는 항상 추천 사유가 있어야 한다.
- 추천 결과에는 주의 조건도 함께 표시한다.
- 실제 모발 상태에 따라 최종 시술안이 달라질 수 있음을 명시한다.
