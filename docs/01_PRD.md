## 1. 프로젝트 개요

- 프로젝트명: 헤어샵 AI 상담 리포트 MVP
- 제품 유형: 헤어샵 태블릿용 B2B AI 상담 보조 솔루션
- 사용 환경: 헤어샵 매장 내 태블릿 가로 화면, 로컬 우선 실행 환경
- 주요 사용자: 고객, 디자이너, 매장 관리자
- 1차 목표: 실제 헤어샵에서 고객 상담 플로우를 테스트하고, 매장 실제 시술 메뉴 기반 추천 리포트가 디자이너 상담과 객단가 향상에 도움이 되는지 검증한다.

---

## 2. 제품 한 문장 정의

헤어샵 고객의 얼굴형과 니즈를 분석해, 해당 매장의 실제 시술 메뉴 중 적합한 조합을 추천하고, 디자이너가 상담에 활용할 수 있는 리포트를 제공하는 태블릿 상담 솔루션.

---

## 3. 해결하려는 문제

### 고객 문제

- 나에게 어떤 헤어스타일이 어울릴지 모름
- 시술 후 결과를 상상하기 어려움
- 디자이너 설명만으로 결정을 망설임
- 커트, 펌, 염색, 클리닉 등 여러 시술 옵션 중 무엇을 선택해야 할지 판단하기 어려움
- 본인의 얼굴형, 얼굴 비율, 이미지 고민을 객관적으로 정리하기 어려움

### 디자이너 문제

- 고객 상담 시간이 길어짐
- 펌, 클리닉, 염색 등 부가 시술 제안이 어려움
- 고객에게 시술 필요성을 설득하기 어려움
- 고객의 고민과 원하는 이미지를 빠르게 파악하기 어려움
- 상담 내용이 디자이너 개인 역량과 경험에 크게 의존함
- 말로만 설명할 경우 고객이 스타일 방향을 이해하지 못하는 경우가 있음

### 매장 문제

- 객단가 상승이 필요함
- 신규 고객 상담 품질을 표준화하기 어려움
- 디자이너별 상담 편차가 큼
- 매장에 등록된 실제 메뉴를 상담 과정에서 자연스럽게 제안하기 어려움
- 고객 상담 데이터를 구조화해 개선에 활용하기 어려움
- 신규 디자이너나 주니어 디자이너의 상담 역량을 보조할 도구가 부족함

- 클레임 방어 가능한 고지 구조 제공

## 5. 주요 사용자

### 고객

- 태블릿에서 원하는 시술 메뉴를 선택한다.
- 현재 헤어 고민을 선택한다.
- 원하는 이미지와 스타일 방향을 선택한다.
- 개인정보 수집 및 촬영 동의 여부를 확인한다.
- 얼굴 촬영을 진행한다.
- 분석 결과와 추천 리포트를 확인한다.
- 디자이너와 상담 후 최종 시술 여부를 결정한다.
- 상담 종료 시 저장 또는 삭제를 선택한다.

### 디자이너

- 고객의 개인정보 동의 여부를 확인한다.
- 고객 얼굴 촬영을 진행하거나 촬영을 보조한다.
- 얼굴 분석 결과와 추천 리포트를 확인한다.
- 고객의 실제 모발 상태, 손상도, 두상, 모질, 기존 시술 이력을 바탕으로 최종 시술안을 조정한다.
- AI 추천 결과를 상담 참고 자료로 활용한다.
- 추천된 매장 메뉴 중 실제로 가능한 시술 조합을 제안한다.
- 고객에게 AI 결과가 실제 시술 결과를 보장하지 않는다는 점을 안내한다.

### 관리자

- 매장 정보를 등록한다.
- 매장 내 실제 시술 메뉴를 등록한다.
- 메뉴별 가격, 소요시간, 설명, 카테고리를 설정한다.
- 메뉴별 추천 조건을 설정한다.
- 디자이너가 상담에 활용할 수 있는 추천 문구를 관리한다.
- 테스트용 상담 데이터를 확인한다.
- 개인정보 저장 및 삭제 정책을 관리한다.

## 6. 핵심 플로우 요약

고객 방문 → 시술 메뉴 선택 → 고민 선택 → 원하는 이미지 선택 → 개인정보 동의 → 얼굴 촬영 → 얼굴 분석 → 추천 리포트 생성 → 디자이너 상담 → 최종 시술안 조정 → 저장/삭제 선택 → 상담 종료

### 상세 플로우

1. 고객이 태블릿에서 상담 시작 버튼을 누른다.
2. 고객이 관심 있는 시술 카테고리를 선택한다.
   - 커트
   - 펌
   - 염색
   - 클리닉
   - 스타일 상담

3. 고객이 현재 고민을 선택한다.
   - 얼굴형 보완
   - 볼륨 부족
   - 손상 모발
   - 넓은 이마
   - 광대 부각
   - 긴 얼굴형
   - 둥근 얼굴형
   - 이미지 변화
   - 관리 편의성

4. 고객이 원하는 이미지 키워드를 선택한다.
   - 자연스러운
   - 세련된
   - 부드러운
   - 어려 보이는
   - 깔끔한
   - 도시적인
   - 고급스러운
   - 개성 있는

5. 고객이 개인정보 및 촬영 동의 내용을 확인한다.
6. 디자이너가 동의 여부를 확인한 뒤 얼굴 촬영을 진행한다.
7. 시스템이 얼굴형, 얼굴 비율, 주요 특징을 분석한다.
8. 시스템이 분석 결과와 고객 선택값을 바탕으로 매장 실제 메뉴 중 적합한 조합을 추천한다.
9. 추천 리포트가 생성된다.
10. 디자이너는 리포트를 참고해 실제 모발 상태와 시술 가능성을 반영한 최종 상담을 진행한다.
11. 고객은 상담 결과 저장 또는 삭제를 선택한다.
12. 저장 동의가 없을 경우 원본 사진과 분석 결과를 삭제한다.

## 7. MVP 성공 기준

- 실제 헤어샵 1곳 이상에서 테스트 가능
- 고객 10~20명 이상 테스트
- 전체 상담 플로우 3분 이내 완료
- 매장 실제 메뉴 기반 추천 가능
- 저장 동의가 없으면 원본 사진과 분석 결과 삭제
- 디자이너가 리포트를 상담에 활용할 수 있음
- 고객이 추천 결과를 이해할 수 있음
- 관리자 화면에서 매장 메뉴를 직접 등록하고 수정할 수 있음
- AI 결과가 상담 참고용이라는 고지가 고객 화면과 리포트에 명확히 표시됨
- 디자이너가 추천 결과를 그대로 확정하지 않고 최종 상담에서 조정할 수 있음

### 정량 기준

- 상담 완료율: 80% 이상
- 상담 플로우 평균 소요 시간: 3분 이내
- 고객 테스트 수: 최소 10명 이상
- 매장 메뉴 등록 수: 최소 10개 이상
- 추천 리포트 생성 성공률: 90% 이상
- 저장 미동의 시 삭제 처리 성공률: 100%

### 정성 기준

- 디자이너가 상담에 활용할 수 있다고 판단하는가
- 고객이 추천 이유를 이해할 수 있는가
- 매장 메뉴 추천이 과도하거나 부자연스럽지 않은가
- AI 이미지 또는 분석 결과가 실제 결과 보장처럼 오해되지 않는가
- 상담 시간이 기존보다 구조화되는가

## 8. 핵심 제한사항

- AI 결과는 실제 시술 결과를 보장하지 않음
- 1차 MVP에서는 정교한 AI 헤어 합성 제외
- 고객 이름/전화번호는 기본 수집하지 않음
- 로컬 우선 구조로 설계
- 원본 사진 저장은 기본값으로 사용하지 않음
- 저장 동의가 없는 경우 상담 종료 후 원본 사진과 분석 결과를 삭제해야 함
- 얼굴 분석 결과는 미용 상담 참고용으로만 사용함
- 의료, 성형, 외모 평가, 관상, 성격 판단 목적의 표현은 사용하지 않음
- 디자이너의 전문 판단을 대체하지 않음
- 추천 결과는 매장에 등록된 실제 시술 메뉴 안에서만 생성함

## 11. 추천 리포트 구성

### 리포트 기본 구성

- 상담 일시
- 선택한 시술 카테고리
- 선택한 고객 고민
- 선택한 이미지 키워드
- 얼굴 분석 요약
- 추천 스타일 방향
- 추천 시술 메뉴
- 추천 이유
- 상담 시 주의사항
- 상담 참고용 고지 문구
- 저장/삭제 상태

### 얼굴 분석 요약 예시

- 얼굴형: 둥근형에 가까움
- 비율 특징: 중안부가 비교적 부드럽게 보이는 편
- 상담 포인트: 옆 볼륨과 앞머리 디자인을 함께 고려하는 것이 좋음
- 추천 방향: 얼굴선을 세로로 정리해주는 스타일 방향 제안

### 추천 시술 메뉴 예시

- 레이어드 커트
- 뿌리 볼륨펌
- 사이드 볼륨 조정
- 톤다운 염색
- 모발 클리닉

### 리포트 고지 문구

본 리포트는 고객 상담을 돕기 위한 참고 자료입니다.
AI 분석 및 추천 결과는 실제 시술 결과를 보장하지 않으며, 최종 스타일과 시술 여부는 디자이너와의 상담 후 결정됩니다.

## 12. 개인정보 및 데이터 처리 원칙

### 기본 원칙

- 고객 이름과 전화번호는 기본적으로 수집하지 않는다.
- 얼굴 사진은 분석 목적 외에 사용하지 않는다.
- 저장 동의가 없는 경우 원본 사진과 분석 결과는 상담 종료 후 삭제한다.
- 저장 동의 여부를 명확히 기록한다.
- 삭제 처리 내역은 삭제 로그로 남긴다.
- 민감한 외모 평가 표현은 사용하지 않는다.

### 저장 가능 데이터

- 상담 세션 ID
- 상담 일시
- 선택한 시술 카테고리
- 선택한 고민
- 선택한 이미지 키워드
- 추천된 매장 메뉴 ID
- 저장 동의 여부
- 삭제 처리 여부
- 상담 리포트 요약

### 저장하면 안 되는 데이터

- 저장 동의 없는 원본 얼굴 사진
- 저장 동의 없는 얼굴 분석 상세 결과
- 고객 이름
- 고객 전화번호
- 주민등록번호 등 고유식별정보
- 의료 정보
- 성형, 질병, 건강 상태와 관련된 추정 정보
- 외모 등급화 또는 점수화 데이터

---

## 13. 클레임 방어 기준

### 반드시 표시해야 하는 문구

- AI 결과는 실제 시술 결과를 보장하지 않습니다.
- 본 리포트는 상담 참고용 자료입니다.
- 최종 시술 여부와 스타일은 디자이너 상담 후 결정됩니다.
- 고객의 모질, 모량, 손상도, 기존 시술 이력에 따라 결과는 달라질 수 있습니다.
- 추천 메뉴는 해당 매장에 등록된 시술 메뉴를 기준으로 제공됩니다.

### 사용하면 안 되는 표현

- 100% 어울리는 스타일
- 실패 없는 헤어 추천
- 사진과 동일한 결과 보장
- 완벽한 얼굴형 분석
- AI가 정확히 진단
- 단점 교정
- 얼굴 결점 분석
- 성형급 변화
- 무조건 예뻐지는 스타일
- 디자이너보다 정확한 AI 추천

## 14. 기술 전제

- 프론트엔드: React
- 언어: TypeScript
- 빌드 도구: Vite
- 스타일링: Tailwind CSS
- 실행 환경: 태블릿 브라우저 또는 PWA

## 16. MVP 검증 질문

- 고객이 3분 이내에 상담 플로우를 완료할 수 있는가?
- 디자이너가 리포트를 실제 상담에 활용할 수 있는가?
- 추천 결과가 매장 실제 메뉴와 자연스럽게 연결되는가?
- 고객이 AI 결과를 실제 결과 보장으로 오해하지 않는가?
- 저장 동의가 없는 경우 데이터가 정상적으로 삭제되는가?
- 관리자 화면에서 메뉴 등록과 수정이 어렵지 않은가?
- 상담 리포트가 객단가 상승 제안에 도움이 되는가?
- 디자이너의 상담 부담을 줄여주는가?
- 고객이 추천 이유를 이해할 수 있는가?
- 실제 헤어샵에서 반복 사용 가능한 흐름인가?
