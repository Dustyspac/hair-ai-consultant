## 1. MVP 정의

1차 MVP는 헤어샵 태블릿에서 고객이 희망 시술과 고민을 선택하고, 얼굴 분석 결과를 바탕으로 매장에 실제 등록된 시술 메뉴를 추천하는 상담 리포트 앱이다.

---

## 2. MVP에 반드시 포함할 기능

### 2.1 고객 기능

| 기능                | 설명                                                    | MVP 필수 여부 |
| ------------------- | ------------------------------------------------------- | ------------- |
| 시작 화면           | 고객이 상담 플로우를 시작하는 첫 화면                   | 필수          |
| 시술 메뉴 다중 선택 | 커트, 펌, 염색, 클리닉 등 희망 시술 선택                | 필수          |
| 고객 고민 선택      | 얼굴형, 볼륨, 손상, 이미지 변화 등 고객 고민 선택       | 필수          |
| 원하는 이미지 선택  | 고객이 원하는 스타일 방향 또는 분위기 선택              | 필수          |
| 개인정보 동의       | 얼굴 촬영, 분석, 저장 여부에 대한 동의 확인             | 필수          |
| 촬영 가이드 확인    | 정면 촬영, 조명, 얼굴 위치 등 안내                      | 필수          |
| 분석 결과 확인      | 얼굴형, 얼굴 비율, 중안부, 광대, 턱선 등 분석 결과 확인 | 필수          |
| 추천 리포트 확인    | 추천 시술 조합과 추천 이유 확인                         | 필수          |
| 저장/삭제 선택      | 상담 결과 및 분석 데이터 저장 또는 삭제 선택            | 필수          |

## 3. MVP에서 제외할 기능

1차 MVP에서는 다음 기능을 개발 범위에서 제외한다.

| 제외 기능                 | 제외 사유                                               |
| ------------------------- | ------------------------------------------------------- |
| 정교한 AI 헤어스타일 합성 | 개발 난이도와 클레임 리스크가 높음                      |
| 실제 3D 얼굴 스캔         | 태블릿 기반 MVP 범위를 초과함                           |
| 자체 AI 모델 학습         | 1인 개발 및 3개월 일정에 부적합함                       |
| 카카오 알림톡 자동 발송   | 외부 연동 및 개인정보 처리 범위가 증가함                |
| 이메일 자동 발송          | 상담 리포트 검증 이후 추가하는 것이 적합함              |
| 고객 전용 앱              | 초기 B2B 상담 리포트 검증과 직접 관련 없음              |
| 다매장 SaaS 관리자        | 파일럿 매장 검증 이후 확장해야 함                       |
| 결제 시스템               | MVP의 핵심 가치 검증과 무관함                           |
| 예약 시스템 연동          | 매장별 예약 시스템 차이로 개발 범위가 커짐              |
| CRM 자동화                | 개인정보 저장 및 운영 부담이 증가함                     |
| 매출 대시보드 고도화      | 초기 파일럿 단계에서는 사용량과 상담 흐름 검증이 우선임 |

## 5. 개발 중 범위 확장 금지 항목

개발 중 다음 항목은 MVP 범위에 포함하지 않는다.

| 금지 항목             | 사유                                                        |
| --------------------- | ----------------------------------------------------------- |
| 완벽한 헤어 합성      | 품질 검수, 클레임 대응, 개발 난이도 모두 높음               |
| 자체 AI 학습          | 데이터셋, 학습 인프라, 품질 관리가 필요해 MVP 범위를 초과함 |
| 고객 앱               | 태블릿 기반 매장 상담 플로우 검증 이후 검토                 |
| 예약/결제/CRM 통합    | 외부 시스템 연동으로 개발 범위가 과도하게 커짐              |
| 앱스토어 정식 배포    | 파일럿 MVP는 웹앱 또는 로컬 실행 기준으로 검증              |
| 모든 태블릿 완벽 대응 | 1차 MVP는 지정된 테스트 태블릿 환경을 우선 지원             |

## 6. MVP 완료 기준

다음 조건을 충족하면 1차 MVP 개발이 완료된 것으로 판단한다.

| 완료 기준                             | 설명                                                                        |
| ------------------------------------- | --------------------------------------------------------------------------- |
| 태블릿에서 전체 상담 플로우 실행 가능 | 시작 화면부터 리포트 확인 및 저장/삭제까지 끊김 없이 진행 가능              |
| 매장 메뉴 등록 가능                   | 관리자 화면에서 실제 시술 메뉴를 등록, 수정, 삭제 가능                      |
| 고객 선택값 저장 가능                 | 희망 시술, 고민, 원하는 이미지, 모발 상태 등 선택값 저장 가능               |
| 얼굴 분석 결과 표시 가능              | 얼굴형, 얼굴 비율, 중안부, 광대, 턱선 등 기본 분석 결과 표시 가능           |
| 추천 메뉴 2~3개 출력 가능             | 매장 메뉴 기반 추천 조합을 2~3개 출력 가능                                  |
| 상담 리포트 출력 가능                 | 고객 선택값, 얼굴 분석 결과, 추천 메뉴, 추천 이유를 리포트 형태로 표시 가능 |
| 삭제 기능 정상 작동                   | 저장 동의가 없는 세션 데이터와 사진이 정상 삭제됨                           |
| 삭제 로그 기록 가능                   | 삭제 처리 시점과 삭제 대상에 대한 로그가 남음                               |

## 7. MVP 핵심 검증 질문

MVP는 다음 질문에 답할 수 있어야 한다.

> 헤어샵 디자이너가 이 상담 리포트를 실제 고객 상담에 사용할 만큼 유용하다고 느끼는가?

세부 검증 질문은 다음과 같다.

- 고객이 태블릿 상담 플로우를 거부감 없이 진행하는가?
- 디자이너가 얼굴 분석 결과와 추천 메뉴를 상담에 활용하는가?
- 추천 리포트가 고객의 시술 결정에 도움을 주는가?
- 매장 메뉴 기반 추천이 실제 객단가 상승 가능성과 연결되는가?
- 고객 사진 저장 없이도 상담 리포트 생성이 가능한가?
- 클레임 방어 문구가 고객과 디자이너 모두에게 명확하게 전달되는가?

## 8. MVP 클레임 방어 기준

MVP의 모든 분석 결과와 추천 리포트에는 다음 원칙을 적용한다.

### 8.1 표현 원칙

- AI 분석 결과는 확정 진단이 아닌 상담 참고 자료로 표현한다.
- 실제 시술 결과를 보장하지 않는다.
- 최종 시술 방향은 디자이너 상담을 통해 결정된다는 문구를 포함한다.
- 고객의 모발 상태, 시술 이력, 두상, 모질, 현장 판단에 따라 결과가 달라질 수 있음을 안내한다.

### 8.2 필수 고지 문구

다음 문구는 개인정보 동의 화면, 분석 결과 화면, 추천 리포트 화면에 포함한다.

```text
본 리포트는 촬영된 이미지와 고객 입력 정보를 바탕으로 생성된 상담 참고 자료입니다.

AI 분석 및 추천 결과는 실제 시술 결과를 보장하지 않으며, 최종 시술 방향은 디자이너와의 상담을 통해 결정됩니다.

추천 스타일과 시술 조합은 고객의 모발 상태, 시술 이력, 두상, 모질, 현장 판단에 따라 달라질 수 있습니다.
```

| -------: | --------------------- | ------------------------------------------------ |
|        1 | 개인정보 동의 및 삭제 | 얼굴 사진 처리와 클레임 방어의 핵심              |

|        5 | 추천 규칙 엔진 v0     | 고객 선택값과 분석 결과를 메뉴 추천으로 연결     |
